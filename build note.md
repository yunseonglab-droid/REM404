# v0.1
## Project Initialization

**Release Date:** Early Development

---

## Overview

REM404 Archive 프로젝트의 첫 번째 시작.

단순히 AR 기술을 사용하는 것이 아니라,

폐건물과 공간에 남겨진 기억을 디지털 공간에서도 이어갈 수 있는
인터랙티브 아카이브 작품을 만드는 것을 목표로 개발을 시작하였다.

이번 버전은 실제 기능 구현보다
프로젝트의 기술적 가능성을 검증하는 것(Proof of Concept)에 집중하였다.

---

# Problem 01

## 사진은 관람 이후 아무것도 남기지 못한다.

### Issue

기존 사진 전시는 관람객이 작품을 감상하고
전시장을 떠나는 순간 경험이 종료된다.

작품은 기억을 이야기하지만

관람객 자신의 기억은 작품 안에 남지 않는다.

관람 이후 작가와 관람객 사이에는
아무런 상호작용도 존재하지 않았다.

### Solution

사진 작품을 시작점으로 하는

인터랙티브 Archive 시스템을 기획하였다.

관람객이 직접 작품에 자신의 기억을 남길 수 있도록
전체 경험 구조를 설계하기 시작하였다.

### Result

REM404 Archive의 핵심 개념이 완성되었다.

'작가의 기억'만 존재하는 작품이 아니라

'관람객의 기억도 함께 축적되는 작품'

이라는 프로젝트 방향이 정해졌다.

---

# Problem 02

## 실제 전시에서 사용할 수 있는 AR 기술 검증

### Issue

프로젝트의 핵심 경험은

사진 작품 위에서 AI 이미지가 자연스럽게 복원되는 것이었다.

하지만

실제 모바일 브라우저에서

안정적으로 이미지를 인식하고

AR을 실행할 수 있는지 확인되지 않은 상태였다.

### Solution

A-Frame과 MindAR을 이용하여

이미지 인식 기반 AR 프로토타입 제작을 시작하였다.

QR 코드 실행부터

이미지 인식까지의 전체 흐름을 테스트하였다.

### Result

모바일 웹 환경에서도

이미지 인식 기반 AR이 가능하다는 것을 확인하였다.

프로젝트를 계속 진행할 수 있는 기술적 기반을 확보하였다.

---

# Problem 03

## 프로젝트 구조 설계

### Issue

초기 단계에서는

프로젝트의 전체 흐름이 존재하지 않았다.

사용자가

언제 QR을 찍고

언제 AR을 보고

언제 기억을 남기는지

전체 경험 구조를 먼저 설계할 필요가 있었다.

### Solution

전시 경험 전체를 설계하였다.

QR 코드

↓

AR 실행

↓

이미지 인식

↓

AI 이미지 복원

↓

기억 남기기

↓

Archive 저장

라는 사용자 경험(User Flow)을 정의하였다.

### Result

REM404 Archive의 전체 UX 구조가 완성되었다.

이후 모든 개발은 이 흐름을 기준으로 진행하게 되었다.

**Release Date:** Early Development

## Overview

v0.2에서는 단순한 AR 이미지 인식을 넘어,

'기억이 천천히 복원된다'는 경험을 전달하는 것에 집중하였다.

# Problem 01

## AR 이미지가 갑자기 나타나는 문제

### Issue

초기 프로토타입에서는

이미지 인식이 완료되는 순간
AI 이미지가 즉시 화면에 나타났다.

기술적으로는 정상적으로 동작했지만,

'기억이 복원된다'는 감정보다는
단순히 이미지가 교체되는 것처럼 느껴졌다.

작품의 서사와 맞지 않는 경험이었다.

### Solution

AI 이미지가

천천히 나타나는 Fade Animation과

Blur Animation을 적용하기 시작하였다.

이미지가 서서히 선명해지는 연출을 통해

기억이 복원되는 과정을 표현하였다.

### Result

단순한 이미지 교체가 아닌

'기억이 복원되는 경험'

이라는 프로젝트의 핵심 연출이 처음으로 구현되었다.

---

# Problem 02

## 사용자에게 현재 상태를 전달하지 못하는 문제

### Issue

사용자는

현재

이미지를 찾고 있는지

복원 중인지

완료되었는지를 알 수 없었다.

상태 변화가 없기 때문에

프로그램이 멈춘 것처럼 느껴질 수 있었다.

### Solution

사용자 안내 UI를 추가하였다.

상황에 따라

- 사진을 맞춰주세요.
- 기억을 찾았습니다.
- 기억을 복원하는 중입니다.

등의 안내 문구를 표시하도록 구성하였다.

### Result

사용자가 현재 진행 상황을 이해할 수 있게 되었으며

AR 경험의 흐름이 훨씬 자연스러워졌다.

---

또한 인식 실패 시 안내 문구가 변경되도록 설계하였다.

### Result

AR 인식 성공률이 향상되었으며

처음 사용하는 사람도
보다 쉽게 작품을 인식할 수 있게 되었다.

# v0.3
## Interactive Archive System

**Release Date:** Development Phase

---

## Overview

v0.3은 REM404 Archive 프로젝트의 가장 큰 전환점이었다.

기존까지의 프로젝트가
AR 기술을 이용하여 AI 이미지를 보여주는 경험에 집중했다면,

이번 버전에서는

관람객이 직접 작품에 참여할 수 있는
Archive 시스템을 구축하는 것을 목표로 개발을 진행하였다.

이번 업데이트를 통해

REM404는

'보는 작품'에서

'함께 만들어가는 작품'

으로 변화하기 시작하였다.

---

# Problem 01

## 작품 경험이 일회성으로 끝나는 문제

### Issue

기존 버전에서는

관람객이

AI 이미지를 감상한 이후

더 이상 작품과 상호작용할 수 있는 요소가 존재하지 않았다.

작품을 감상하는 순간 경험이 종료되었으며,

관람객 자신의 기억은 작품 안에 남지 않았다.

### Solution

Archive 시스템을 새롭게 설계하였다.

관람객이 작품을 감상한 이후

직접 자신의 기억을 입력하고

그 기억을 작품 안에 영구적으로 저장할 수 있도록 개발하였다.

Firebase를 이용하여

실시간 Archive Database를 구축하였다.

### Result

작품이 단순한 감상 대상이 아니라

관람객의 참여를 통해

계속 성장하는 Archive 형태로 발전하였다.

---

# Problem 02

## 기억이 저장되지 않는 문제

### Issue

사용자가 입력한 기억을

실제로 저장할 수 있는 구조가 존재하지 않았다.

새로고침 이후에는

모든 정보가 사라지는 상태였다.

### Solution

Firebase Firestore를 프로젝트에 도입하였다.

기억을 입력하면

Database에 즉시 저장되도록 구현하였다.

또한

기억 저장 시

프로젝트 정보

버전

언어

생성 시간

등의 메타데이터도 함께 저장하도록 설계하였다.

### Result

REM404 Archive의 핵심 기능인

Memory Archive가 처음으로 완성되었다.

---

# Problem 03

## 사용자에게 저장 결과를 전달하지 못하는 문제

### Issue

기억을 저장하더라도

사용자는

정상적으로 저장되었는지

확인할 수 없었다.

### Solution

Archive Complete 화면을 새롭게 제작하였다.

기억 저장 이후

감사 메시지와 함께

현재 Archive에 저장된 기억의 개수를 표시하도록 구성하였다.

Firebase Count 기능을 이용하여

실시간 Archive 규모를 확인할 수 있도록 구현하였다.

### Result

사용자는

자신의 기억이 실제 Archive의 일부가 되었다는 것을

직접 확인할 수 있게 되었다.

---

# Problem 04

## 기억 입력 UX

### Issue

기억을 입력하는 과정이

작품 경험과 자연스럽게 연결되지 않았다.

입력 과정이

단순한 Form처럼 느껴졌다.

### Solution

Archive UI를 새롭게 디자인하였다.

- 기억 입력 화면
- 글자 수 제한
- 저장 버튼
- 저장 완료 화면

전체 흐름을 하나의 경험으로 구성하였다.

또한

입력 글자 수를 실시간으로 표시하여

사용자가 쉽게 작성할 수 있도록 개선하였다.

### Result

기억을 남기는 과정 자체가

작품 경험의 일부가 되었다.

# Problem 01

## AI 복원 이후에도 작품을 계속 비춰야 하는 문제

### Issue

AI 이미지가 완전히 복원된 이후에도
카메라가 전시 사진에서 벗어나면 모든 과정이 처음부터 다시 시작되었다.

관람객은 기억을 읽거나 버튼을 누르기 위해서도
계속 작품을 비추고 있어야 했으며,
이는 실제 전시 환경에서 매우 불편한 경험을 만들었다.

### Solution

Recognition Experience Lock 시스템을 추가하였다.

AI 이미지가 완전히 복원되는 순간부터
AR Tracking 상태와 관계없이 현재 경험을 유지하도록 변경하였다.

### Result

- 작품을 계속 비출 필요가 없어졌다.
- 관람객이 자연스럽게 휴대폰을 들고 다음 경험으로 이동할 수 있게 되었다.
- 실제 전시 UX가 크게 향상되었다.

# Development Retrospective

REM404는 이제 단순한 AR 기술 시연이 아니라,

> **한 사람이 남긴 기억이 또 다른 사람의 기억으로 이어지는 디지털 아카이브**

라는 명확한 작품 구조를 갖춘 프로젝트로 발전했다.

이 방향성은 앞으로의 졸업작품과 전시에서도 프로젝트의 핵심 축이 될 것이다.
